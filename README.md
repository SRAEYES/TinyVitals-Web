# TinyVitals — Track. Grow. Protect.

Premium AI-powered pediatric health management platform marketing website. Built with Apple-level design, Stripe-grade UX, and Raycast-inspired interactivity.

![Next.js](https://img.shields.io/badge/Next.js-16.2.6-black) ![React](https://img.shields.io/badge/React-19-blue) ![TypeScript](https://img.shields.io/badge/TypeScript-5-blue)

## 🎨 Design Philosophy

This website embodies premium SaaS design inspired by Apple.com, Linear.app, Stripe, Raycast, Arc Browser, Vercel, and Nothing.tech.

## ✨ Features

### 12 Premium Sections

1. **Hero Section** - Cinematic landing with animated DNA ribbon
2. **Scroll Storytelling** - Apple-style parallax narrative
3. **Feature Showcase** - 7 key features with glass morphism
4. **Health Sphere** - 3D floating visualization
5. **How It Works** - 6-step animated timeline
6. **Trust & Privacy** - Security-focused design
7. **Values** - 5 core mission values
8. **Statistics** - Animated counters
9. **App Gallery** - 3D carousel
10. **Team** - Meet the team
11. **Testimonials** - Infinite marquee
12. **Final CTA** - Call-to-action
13. **Footer** - Professional footer

## 🛠 Tech Stack

- **Frontend**: Next.js 16 | React 19 | TypeScript | Tailwind CSS 4
- **Animation**: GSAP | Framer Motion | Lenis
- **3D**: Three.js | React Three Fiber
- **Utilities**: next-themes | Lucide Icons

## Getting Started

First, run the development server:

```bash
npm run dev
# or
yarn dev
# or
pnpm dev
# or
bun dev
```

Open [http://localhost:3000](http://localhost:3000) with your browser to see the result.

## Build & Production

```bash
# Build for production
npm run build

# Start production server
npm start

# Lint code
npm run lint
```

## 🚀 Deployment

### GitHub Setup

1. Create repository at https://github.com/new
2. Run these commands:

```bash
git remote add origin https://github.com/sraeyes/tinyvitals-web.git
git branch -M main
git push -u origin main
```

### Vercel Deployment

1. Go to https://vercel.com/new
2. Connect your GitHub repository
3. Click Deploy

See [DEPLOYMENT_GUIDE.md](./DEPLOYMENT_GUIDE.md) for detailed instructions.
